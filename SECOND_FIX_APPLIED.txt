NEW ISSUE: Airflow 2.8+ doesn't allow in-memory SQLite

ERROR:
  AirflowConfigException: Cannot use relative path: `sqlite:///:memory:`
  to connect to sqlite. Please use absolute path such as 
  `sqlite:////tmp/airflow.db`.

ROOT CAUSE:
  Airflow 2.8+ added a validation that requires absolute paths for SQLite
  databases. The `:memory:` special path is no longer allowed.

FIX APPLIED:

1. ✅ Updated tests/conftest.py
   Changed from: sqlite:///:memory:
   Changed to:   sqlite:////tmp/airflow_test.db
   
   Uses tempfile.gettempdir() for cross-platform compatibility

2. ✅ Updated .github/workflows/airflow-integration.yml  
   Set AIRFLOW__DATABASE__SQL_ALCHEMY_CONN=sqlite:////tmp/airflow_test.db

CODE CHANGE IN conftest.py:
```python
# Before (didn't work with Airflow 2.8+)
os.environ.setdefault('AIRFLOW__DATABASE__SQL_ALCHEMY_CONN', 'sqlite:///:memory:')

# After (works with Airflow 2.8+)
AIRFLOW_DB_PATH = os.path.join(tempfile.gettempdir(), 'airflow_test.db')
os.environ.setdefault('AIRFLOW__DATABASE__SQL_ALCHEMY_CONN', f'sqlite:///{AIRFLOW_DB_PATH}')
```
